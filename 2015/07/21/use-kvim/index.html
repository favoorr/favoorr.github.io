Centos6.6安装使用k-vim

发表于 2015-07-21   |   分类于 vim   |     |   阅读次数

环境

centos 6.6 64 位

没敢在自己的 mac 上直接撸，怕把 mac 上的 vim 搞残了 …

新手配置vim比较困难，先暂且使用别人的 vim

曾经试着参考别人教程，搞一个属于自己的 vim，可惜 c c++ 基础太弱，一直没实现

正文开始，参考 k-vim readme，和自己跳的坑

1.从 github 克隆到本地

git clone https://github.com/wklken/k-vim.git

2.安装依赖包

# ctags, ag(the_silver_searcher)

2.1 系统依赖

yum install python-devel.x86_64 or yum info python-devel.x86_64

yum groupinstall 'Development Tools'

rpm -Uvh http://download.fedoraproject.org/pub/epel/7/x86_64/e/epel-release-7-5.noarch.rpm

yum install the_silver_searcher 如果是centos6，需要自己安装，会出现错误 错误：xz compression not available 

需要执行 rm -rf /etc/yum.repos.d/epel* ，使 yum 恢复ok

centos6.6 安装 the_silver_searcher 参考 ，我写的教程

2.2 使用 python

pip install pyflakes
pip install pylint
pip install pep8

要安装很多东西 pip ，参考

2.3 如果使用Javascript, 不需要的跳过

# 安装jshint和jslint,用于javascript语法检查
# 需要nodejs支持,各个系统安装见文档 https://github.com/joyent/node/wiki/Installing-Node.js-via-package-manager

# todo 没用 javascript 暂时跳过

3 安装

3.1 进入目录

一执行，就是好多个小时，至少5小时，具体我也不知道，还跟个人网速有关，我一整晚电脑放公司没关机 ….

安装完成后，第二天来看到有错误，没理会，很想知道 k-vim 安装完成后是个什么样，vim 是残了还是变高大上了

于是执行

具体经历如下：

错误一：一直提示

成功编译 ycm 之后，上面提示从此不再出现

错误二：

奇怪的是这个错误只出现一次，Press 任意键进入 vim 后，再也没有继续出现该错误

然并卵

我之前自己参考过 https://github.com/yangyangwithgnu/use_vim_as_ide 里面的 readme 撸过 vim，然后配置 colorscheme solarized 想自己搞个主题，

但是哪，我按照教程没撸出来，也就暂时放弃了，于是很想看看作者是如何搞的，我是 vim 新手，就看过 陈浩 的教程，自己能活下来 …

我是直接给root安装的vim，所以，我就去 /root/.vim/ 目录里面看了下，竟然没有 colors 目录，我于是自己新建目录，scp 上传了自己上次下载的 solarized.vim ，结果上面这个不在出现的错误竟然又开始出现了，并且，注意是并且它还带来了新错误

和一堆自己上次搞 主题时候碰到的一样的错误 ，还有 k-vim 作者更新到了 9.0 ，安装好之后的目录下 vim ~/.vim/vimrc ，Version: 8.1 没有更新，赶紧删除 新建的 colors 目录，让世界恢复平静

有此看到，错误二 还是必须要解决，一个个开始 google

查了资料，发现问题解决并没有那么容易，我得先分解下 k-vim 作者这 30++ 插件

记一

准备在 mac 上使用 k-vim 了，

brew install vim ，brew 里面的版本已经 404 了，需要自己手工修改

记二

mac 自带了 vim，新安装了vim之后，没有必要必须删除老版本，在使用的 shell 里面设置即可
