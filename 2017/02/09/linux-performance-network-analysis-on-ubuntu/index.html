以 Ubuntu 为例列举 Network 性能分析常用命令

发表于 2017-02-09   |   分类于 performance   |     |   阅读次数

操作系统版本 Ubuntu 16.04.1

手工实践记录，把新知识手工敲一遍，是把知识变成自己的第一步

网络-分析

netstat

多种网络栈和接口统计信息

默认 列出连接的套接字
-a 列出所有套接字的信息
-s 网络栈统计信息
-i 网络接口信息
-r 列出路由表

其他选项能修改输出，例如

-n 不解析 ip 地址为主机名
-v 显示冗长的详细信息

Iface 网络接口
MTU 最大传输单元
RX 接收
TX 传输
OK 成功传输的数据包
ERR 错误数据包
DRP 丢包
OVR 超限

丢包和超限是网络接口饱和的指针

-c 连续模式能与 -i 一并使用，每秒输出这些累积的计数器，它提供技术数据包速率的数据

网络栈统计数据

输出列出了多项按协议分组的网络数据主要是来自 TCP 的

sar

统计信息历史

可以观测当前活动病情能配置为保存和报告历史统计数据

rxpck/s
txpck/s
rxkB/s
txkB/s

rxcmp/s

rxerr/s 接收数据包错误
txerr/s 传输数据包错误
coll/s 碰撞
rxdrop/s 接收丢包 缓冲满
txdrop/s 传输数据包丢包 缓冲满
rxfifo/s 接收的数据包 FIFO 超限错误
txfifo/s 传输的数据包 FIFO 超限错误

irec/s
fwddgm/s
orq/s

idisc/s
odisc/s

active/s 新的主动 TCP 连接 connect()
passive/s 新的被动 TCP 连接 listen()
iseg/s 输入的段（接收）
oseg/s 输出的段 (传输)

retrans/s TCP 段重传

totsck 使用中的总数据包 sockets
ip-frag 当前队列中的 IP 数据片 fragments
tcp-twTIME-WAIT 的 TCP 套接字 sockets

许多统计信息包括方向和统计单位

rx 是接收
i 输入
seg 是段

还可以统计 ICMP UDP NFS 和 IPv6 在内的信息
IpInReceives = irec/s

ifconfig

接口配置

列出所有接口当前的配置，与 netstat -i 一致
txqueuelen 是这个接口发送队列的长度

linux 下的 ifconfig 命令逐渐被 ip 命令替代

ip

网络接口统计信息

能够配置网络接口和路由，并且观测他们的状态和统计信息。

nicstat

网络接口吞吐量和使用率

-z 用来忽略数值为 0 的接口
-t 显示 TCP 统计信息

最前面是系统启动以来的总结，紧接着是按时间间隔的总结

Int 接口名称
%Util 最大使用率
Sat 反应接口饱和度的统计信息
r 读（接受）
w 写 （传输）

KB/s 千字节每秒
PK/s 数据包每秒
Avs/s 平均数据包每秒，以字节为单位

ping

测试网络连通性

rtt 每个包的往返时间
由于 时间戳是 ping 命令自己计量的，其中包括获取时间戳到处理网络 IO 的整个 CPU 代码路径执行时间

traceroute

测试网络路由

发出一系列数据包实验性地探测到一个主机当前的路由
它的实现利用了递增每个数据包 IP 协议的生存时间 TTL，
从而导致网关顺序的发送 ICMP 超时响应报文，向主机揭示自己的存在（如果防火墙没有拦截他们）

每一跳显示连续的三个 RTT，它们可用作网络延时统计信息的粗略数据源。
类似 ping 由于发送低优先级的数据包，它可能显示出比其他应用程序协议更高的延时

也可以把显示的路径作为静态性能调优的研究对象。网络被设计为动态的并且能响应故障。路径的变化可能会降低性能。

pathchar

确定网络路径特征

pathchar 类似于 traceroute ，并且包括了每一跳间的贷款，它的实现是利用了多次重复发送一系列不同长度的网络数据包然后统计分析它们

在 ubuntu 上，pathchar 叫 pchar

依据跳跃数的不同，可能需要数分钟

tcpdump

网络数据包嗅探器

可以捕捉并检查网络数据包。它输出网络包信息到 STDOUT，或者把数据包写入文件以供稍后的分析。后者通常更实用：过高的数据包速率导致了不能实时研究他们

将 eth4 的数据写入 /tmp 下的文件

每一行显示数据包时间（精度到毫秒），它的源和目标 IP 地址，以及 TCP 报头值。

-n 禁用 IP 地址解析为主机名
-v 打印包冗长的细节
-e 链路层报头
-x -X 十六进制地址转储

在性能分析过程中，把时间戳列改为显示数据包间的时间差 -ttt ，或者自第一个数据包以来的时间 -ttttt 会更有帮助

就 CPU 成本和存储而言，捕捉数据包是昂贵的，在可能的情况下尽量短时间使用 tcpdump 以限制其对性能的影响

wireshark

图形化网络数据包检查器

DTrace

TCP/IP 栈跟踪，连接、数据包、丢包、延时

perf

TCP/IP 栈跟踪，连接、数据包、丢包、延时

其他工具

strace 跟踪套接字相关的系统调用并检查其使用的选项，使用系统开销较高
lsof 按进程 id 列出包括套接字细节在内的打开的文件
ss 套接字统计信息
nfsstat nfs 服务器的和客户机统计信息
iftop 按主机嗅探总结网络接口吞吐量
/proc/net 包含许多网络统计信息文件

试验工具

iperf

iperf 是一款测试最大 tcp 和 udp 吞吐量的开源工具，必须同时在服务器和客户机上运行

服务器运行

将套接字缓冲长度由默认的 8KB 增加到 128KB (-1 128k)

客户机运行

最后会给一个总的汇总

-c host 连接到主机名或者 IP 地址
-1 128k 使用 128KB 套接字缓冲
-P 2 运行于两个客户机线程的并行模式
-i 1 每秒打印时间间隔总结
-t 60 总测试时间 60s

测试完成后，可以在 server 端看到结果
