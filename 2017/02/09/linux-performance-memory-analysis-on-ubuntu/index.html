以 Ubuntu 为例列举 Memory 性能分析常用命令

发表于 2017-02-09   |   分类于 performance   |     |   阅读次数

操作系统版本 Ubuntu 16.04.1

手工实践记录，把新知识手工敲一遍，是把知识变成自己的第一步

内存分析工具

vmstat

虚拟和物理存储器统计信息
虚拟内存统计信息指令。提供了包括当前内存和换页在内的系统内存健康程度总览。
CPU 统计信息页包含在内

该版本的 vmstat 不在第一行输出自系统启动至今的 memory 列的汇总数据，而立刻显示当前状态。

默认数据列如下，单位为 KB

swpd 交换出的内存量
free 空闲的可用内存
buff 用于缓冲缓存的内存
cache 用于页缓存的内存
si 换入的内存 （换页）
so 换出的内存 （换页）

如果 si 和 so 一直非 0，那么系统郑村镇内存压力并换页到交换设备或文件，用其他工具可以研究什么在消耗内存，例如能观察每进程内存使用的工具

当服务器拥有大量内存是，数据列会不对齐而影响阅读。可以试着用 -Sm 来修改输出单位为 MB

选项 -a 可以输出非活动和活动页缓存的明细

sar

历史统计信息

-B 换页统计信息
-H 大页面统计信息
-r 内存使用率
-R 内存统计信息
-S 内存空间统计信息
-W 交换统计信息

pg 表示页
kb 表示 KB
% 表示百分比
/s 表示每秒

详细部分可读源码 mm/vmscan.c

%vmeff 是衡量页回收率的一个有趣指标。高数值意味着成功从非活动列表中回收了页，表示健康
低数值意味着系统在挣扎。 100% 是高数值，少于 30% 是低数值

slabtop

内核块分配统计信息

可以通过 slab 分配器输出内核 slab 缓存使用情况。类似 top 实时更新屏幕

输出包括顶部的汇总和 slab 列表，

OBJS 对象数量
ACTIVE 多少是活动的
USE 使用百分比
OBJ SIZE 对象大小
CACHE SIZE 缓存大小

sc 按缓存大小排序，最大值在顶端
slab 统计信息取自 /proc/slabinfo, 也可以用 vmstat -m 输出

ps

进程状态命令 ps 可以列出包括内存使用统计信息在内的所有进程细节

%MEM 主存使用（物理内存、RSS）占总内存的百分比
RSS 常驻集合大小 KB
VSZ 虚拟内存大小

RSS 显示主存使用，它也包括如系统库在内的映射共享段，可能会被几十个进程共享。如果把 RSS 列求和，可能会发现它超过系统的内存综合，这是由于重复计算了这部分共享内存。分析共享内存的使用可参考 pmap 命令

linux 下也可以输出 maj_flt min_flt 列

maj_flt 严重缺页
min_flt 轻微缺页

top

监视每个进程存储器使用率

监控排名靠前的运行中进程并且显示内存使用统计信息

Mem 主存
Swap 交换内存
buffers 缓冲缓存
cached 页缓存

VIRT 虚拟内存
RES 主存，常驻集合大小
%MEM 占用主存占比

pmap

进程地址空间统计信息

列出进程的内存映射，显示它们的大小、权限及映射的对象。这使进程内存使用能被仔细的检查以及量化共享内存。

Kbytes 虚拟内存
RSS 主内存
Dirty 私有匿名内存
mode 权限

很少有内存是匿名的，而且大部分是只读的 r-x，这意味着这些页面可以被其他进程共享，尤其是系统库。

对于有很多映射的进程，pmap 输出可能会很长。它还会暂停进程以报告内存使用，这会影响任务的性能。应在需要针对和分析时运行，不应作为监视工具频繁运行

DTrace

分配跟踪

DTrace 能跟踪用户和内核级的内存分配、轻微和严重缺页以及页面换出守护进程运行。这些功能支持使用特征归纳和想下挖掘分析。

其他工具

free 报告空闲内存，包括缓冲区高速缓存和页缓存
dmesg 检查来自 OOM 终结者的 Out of memory 信息
valgrind 一个包括 memcheck 在内的性能分析套件，它是一个内存使用分析的封装程序，可用于发现泄露。它能造成严重的系统开销，可能引起目标系统慢 20 - 30 倍
swapon 添加和观察物理交换设备或者文件
iostat 如果交换设备是物理磁盘或块，设备 I/O 可以用 iostat 来观测，它能指出系统是否在换页
perf 能观察 CPI MMU TSB 事件，以及源自 CPU 性能计数器的内存总线的停滞周期计数。还提供缺页以及一些内核内存 kmem 事件的探针
/proc/zoneinfo 内存区域的统计信息
/proc/buddyinfo 内核页面伙伴分配器统计信息
