在sublime 中新建snippet

在使用markdown编写博客之后，开头都要引入模板套用
这时候代码片段snippet就派上用场了，有Eclipse使用的
经验，然后我想这个工具肯定是有这个功能的，然后就直接扑上去找，

新建snippet

菜单栏 Tools –> new Snippet 就是它了,点击之后会出现如下模板

<snippet>
    <content><![CDATA[
Hello, ${1:this} is a ${2:snippet}.
]]></content>
    <!-- Optional: Set a tabTrigger to define how to trigger the snippet -->
    <!-- <tabTrigger>hello</tabTrigger> -->
    <!-- Optional: Set a scope to limit where the snippet will trigger -->
    <!-- <scope>source.python</scope> -->
</snippet>

各个节点解释如下

1.新建的模板必须包括在 <![CDATA[ ]]>中，即xml中得文本数据

2.新建时，snippet里面的缩进会保留到引用的位置

3.hello节点表示输入hello后，摁 Tab 键，会调用snippet

4.source.python ，表示该snippet只能在python源码中引用，其他还有很多，
详细需要了解Sublime的scope，而且如果这里填写错误，snippet会调取失败

5.${1:this} ${2:snippet}，表示调出snippet后，是否在指定位置填写默认值和光标的位置，
即会在位置1显示this位置二显示 snippet，而且在光标在位置一时，摁Tab会跳转到位置二

6.md-post 表示在点击 Tools –> snippet 后，显示的描述

7.新建后会保存到 /Users/User/Library/Application Support/Sublime Text 3/Packages/User/ 路径
文件命名示例：md-post.sublime-snippet

使用注意点

保存文件后，不需要重启Sublime 3 ，直接立马使用

在第一次新建的时候，输入了错误的scope，一直不成功，详细scope要自行了解下

示例

<snippet>
    <content>
    <![CDATA[
---
layout: post
title:  "${1}"
date:  ${2}
categories:  
tags: 
---
    ]]>
    </content>
    <tabTrigger>post</tabTrigger>
    <description>md-post</description>
</snippet>

保存文件为 /Users/User/Library/Application Support/Sublime Text 3/Packages/User/md-post.sublime-snippet

编写博客时，输入post，然后Tab 就可以调用snippet
