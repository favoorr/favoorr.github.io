git rebase命令学习

阮老师对git rebase讲了一个点，可以和merge类似的操作，
最基本的时候，可以达到merge相同效果

一个台湾同胞写的教程，非常详细

http://blog.yorkxin.org/posts/2011/07/29/git-rebase

这里也简单介绍到了

http://gitbook.liuhui998.com/4_3.html

写下自己的理解：

假设

三个分支 master dev favoorr

场景一

favoorr衍生于 dev ，现在想修改到master，

可以在favoorr分支执行 git rebase master ，切换参考基准

场景二

favoorr 分支,本来衍生自提交aaaaa，现在想修改，衍生自提交yyyyy

在favoorr分支上上执行 git rebase –onto yyyyy aaaaa

git rebase –onto

git rebase –continue 如果有冲突，处理完冲突后，继续

场景三

1.开发的时候，遇到很多提交，想合并连续两次的小提交

2.或者之前的提交想改变顺序，提交zzz，应该放到提交xxx的前面

3.或者之前的提交commit写的有错别字，那么可以用rebase修改

git rebase -i xxxxx,你准备从这里开始整理提交，在提交一遍

git rebase 都可以做到

svn 里面没有搞过这种修改

git操作远程仓库

发表于 2015-01-08   |   分类于 git   |     |   阅读次数

git操作远程仓库

这里对fetch和rebase的用法存在疑惑，Google了下，找到阮老师的教程，
推荐看原版，下面内容加入了自己理解

git clone

git clone https://github.com/jquery/jquery.git 远程主机被默认命名为 origin（起源）
git clone https://github.com/jquery/jquery.git myjquerycopy 创建自己的目录，一般不需要加目录

支持的所有的协议

git clone http[s]://example.com/path/to/repo.git/
git clone ssh://example.com/path/to/repo.git/
git clone git://example.com/path/to/repo.git/
git clone /opt/git/project.git 
git clone file:///opt/git/project.git
git clone ftp[s]://example.com/path/to/repo.git/
git clone rsync://example.com/path/to/repo.git/

git remote

git remote 查看所有的远程主机，只会显示列表
git remote -v
git clone -o jQuery https://github.com/jquery/jquery.git  将远程仓库命名为jQuery，即更改默认的origin
git remote show 查看更加详细的信息
git remote add origin https://github.com/jquery/jquery.git 添加远程仓库
git remote rm origin 删除远程仓库
git remote rename origin github 将远程仓库由默认的origin更改为github

git fetch

这个命令是将远程仓库的更新拉到本地仓库，而不是本地工作区，但不会与当前分支合并

git fetch 拉取远程仓库所有分支的所有更新
git fetch master 仅仅拉取master分支的更新
git branch -r 查看远程的所有分支
git branch -a 查看所有分支
git checkout -b newBrach origin/master 取回远程分支的更新，并创建新分支
git merge origin/master 将远程分支更新取回本地仓库，并与当前分支合并

git rebase origin/master 
等同于
git merge origin/master

git pull

取回远程主机的某个分支与本地的某个分支合并

git pull <远程主机名> <远程分支名>:<本地分支名>

git pull origin next:master 取回远程主机的next分支与本地的master分支合并
git pull origin next 取回远程主机的next分支，与当前默认分支合并，这里要注意了

git pull origin next 
等同于 
git fetch origin
git merge origin/next

本地分支与远程同名分支默认会建立追踪关系，git clone 时，会将本地master和远程master建立关系

git branch --set-upstream master origin/next 将本地的master和远程的next建立追踪关系

如何查看本地分支是否与远程分支建立追踪关系？？？？

git branch -r 可以列出远程有追踪关系的分支，但是不能针对分支查看，比如查看本地dev关联什么分支，可能习惯上，本地分支和远程分支名称都要一样，你可以不关联，但是不要搞本地 xx 分支 和远程yy分支关联

如果当前分支有与之建立追踪关系的远程分支，git pull可以省略远程分支名

git pull origin 本地分支自动拉取origin与之建立追踪关系的分支的更新并且合并

git pull 如果只有一个远程主机，并建立一个追踪关系，可以省略全部参数

git pull --rebase <远程主机名> <远程分支名>:<本地分支名> 如果需要采用rebase模式，何时采用哪？？？

git push

将本地分支的更新推送到远程分支

git push <远程主机名> <本地分支名>:<远程分支名>

git push origin master 如果已经建立追踪关系，可以省略远程分支，如果远程分支不存在，就会新建

git push origin :master 省略本地分支，等于推送了空到远程分支，相当于删除所有内容
等同于
git push origin --delete master 删除origin主机的master分支

git push origin 如果本地分支和远程分支有追踪关系，两个名词都可以省略

git push 如果只有一个远程追踪，远程主机也可以省略

git push -u origin master 如果存在多个远程主机关联，可以使用-u设置默认主机，设置后，可以使用 git push

push.default 含义，2.0之前采用matching，之后改为simple

git config --global push.default matching
git config --global push.default simple

simple 默认只推送当前分支
matching 会推送所有有追踪关系的本地分支

git push --all origin 不管是否有对应的远程分支，全部推送到远程主机

git push --force origin 如果远程高于本地无法推送，可以强行推送，不建议使用

git push origin --tags tag默认不会推送，必须加参数

git push --set-upstream origin master  和远程master分支建立追踪关系，并push当前分支

新增远程仓库操作

git clone -o booyah  命名初始远程仓库不叫 origin ，叫 booyah

git fetch origin  更新本地分支与远程分支同步，拉取远程代码到本地

git remote add github git@xxxx.xxx.xx:yy.git 添加新的远程仓库 github 

git push origin serverfix:awesomebranch 将本地分支 serverfix 推送到远程仓库 origin 中 awesomebranch

git merge origin/serverfix 将远程分支与自己当前分支合并

git checkout -b serverfix origin/serverfix  新建分支，以 远程分支为基础

git checkout -b sf origin/serverfix  远程分支和本地分支使用不同的名字

git checkout --track origin/serverfix  跟踪分支，远程分支和本地分支操作简化命令

git branch -u origin/serverfix 当前本地分支关联远程分支 --set-upstream-to 的简写 -u

git merge @{u}  等于 git merge git merge origin/master  ,使用 @{u} @{upstream} 代替 origin/master

git fetch --all; git branch -vv   拉取最新，并且显示本地分支与远程分支情况

单机功能

git init 初始化本地仓库
git add filename ... 添加到暂存区 

git commit -m "a tip" 添加到本地仓库 没有add的，不会 commit
git commit readme.txt -m "test1"   会提交所有的改动，即使没有add，

git status 查看本地工作区状态
git log 查看本地提交日志
git reflog 查看本地所有操作记录
git reset --hard xxxxx 从本地仓库恢复到某个版本，改变head指针
git reset --hard HEAD^  恢复上一个稳定版本

log的几种用法 截取的zsh下的alias  ，alias | grep 'git log'
glg='git log --stat --max-count=10'
glgg='git log --graph --max-count=10'
glgga='git log --graph --decorate --all'
glo='git log --oneline --decorate --color'
glog='git log --oneline --decorate --color --graph'

git log --pretty=oneline --abbrev-commit   简短显示提交内容
git log -p -2 展示每次提交的差异,显示的是行层面的对比
git log -U1 --word-diff  显示的是单词层面的对比
git log --stat  显示增改行数统计
git log --pretty=oneline 一行显示  glo ,会显示最后一次提交的

gitk 一个查看git发布的工具，执行命令后，直接显示

概念部分

工作区 新建文件，就在工作区
暂存区 通过add 就添加到暂存区
本地仓库 通过commit，就添加到本地仓库，进行版本管理

git 管理的是提交，而不是文件，下面是示例

git对同一个文件修改两次，但是只有一次会被提交到仓库

第一次修改 – add – 第二次修改 – commit – 只有第一次add的被commit到本地库

上面操作过后，第二次对文件的修改并没有被提交

撤销修改

git checkout -- readme.txt  恢复一次修改

一种可能是还没有add到暂存区，那么直接可以恢复到修改之前

一种是add到暂存区，还没有到本地仓库,然后又修改了，这样子会回到上一次add的状态，没有add的会回退

会回到上次 add 之前的状态

git reset HEAD  file 回到 add 前的状态，从暂存区拉倒工作区

git checkout -- file 从工作区拉倒修改前

删除文件

git rm file  删除文件，否则不能提交

分支

分支使用：创建分支--添加功能--分支提交--切换主分支--主分支合并--删除子分支
git checkout -b dev  创建并切换
git branch dev 创建dev分支
git checkout dev  切换dev分支
vim readme.txt
git add readme.txt
git commit -m "branch test"
git checkout master
git merge dev
git branch -d dev

当出现同时编辑一行的时候，会出现冲突，冲突后需要手工合并，在提交

合并分支中，

git merge --no-ff -m "merge with no-ff" dev

通过参数，禁用fast-forward,产生合并记录

git log --graph --pretty=oneline --abbrev-commit  查看分支合并图

分支使用场景：暂停手头工作–拉取分支–修改bug–提交–合并

工作进行到一半，需要修改bug，

git stash 

fix bug

git merge

git stash pop

git stash list

git stash apply 

对于没有合并的分支

git branch -D branchName 强行删除

标签tag

git tag v1.0  为当前提交打tag 1.0
git tag 列出当前有的 tag
git show v0.9 显示某个标签信息
git tag -a v0.1 -m "version 0.1 released" 3628164  从以往的提交建立带说明的标签
git tag -s v0.2 -m "signed version 0.2 released" fec145a 私钥签名的标签

git push origin --tags  一次性将本地未推送的标签全部推送

git tag -d v0.9  删除本地
git push origin :refs/tags/v0.9 删除远程标签

git push origin dev  将本地仓库内容推送到远程dev分支，如果远程分支版本高于自己，会失败，要先拉取，合并，在push
