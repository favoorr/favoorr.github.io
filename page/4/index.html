git 账号配置多个不同邮箱

之前使用git仅限于提交到 github ，其他平台如 gitlab bitbucket 目前没有使用，
现在公司私服 gitlab 需要添加独立的邮箱和用户名，要不然每次提交都是自己
Gmail的地址

1.生成公司 gitlab 私服的 ssh key

ssh-keygen -t rsa -f ~/.ssh/id_rsa_yourdomain_gitlab -C "qingyun@xxx.com"

2.ssh 服务默认制度去 id_rsa 默认的，现在需要添加让它认识 id_rsa_yourdomain_gitlab

ssh-add ~/.ssh/id_rsa_yourdomain_gitlab

3.新建本地认证文件 ~/.ssh/config

vim ~/.ssh/config

Host github.com
    HostName github.com
    User favoorr
    IdentityFile ~/.ssh/id_rsa

Host dev1
    HostName dev1
    User youname
    IdentityFile ~/.ssh/id_rsa_yourdomain_gitlab

4.取消全局配置

git config --global --unset user.name
git config --global --unset user.email

5.项目单独配置 name 和 email

git config user.name "youname"
git config user.email "qingyun@yourdomain.com"

如果不设置user.name 和 user.email 就不能使用，必须指定用户名和邮箱

6.新问题，ssh-add 之后，每次重启mac，重现执行ssh-add -l都是 提示 the agent has no identities

Google查资料后，找到这篇文章 是否必须每次添加ssh-add

需要使用

ssh-add -K  ~/.ssh/id_rsa_yourdomain_gitlab

nginx 配置

gitlab omnibus 版本内置了一个nginx，可以通过 nginx[‘enable’] = true 启用禁用

external_url 这个链接的配置会影响内置 nginx 启动监听的端口

external_url 'http://192.168.143.129'  默认监听 80 端口，直接输入ip就可以访问

external_url 'http://192.168.143.129:9002'  监听 9002 端口，如果修改了端口，需要手工将之前的nginx进程干掉

安装路径

/opt/gitlab gitlab和依赖应用的代码
/var/opt/gitlab gitlab-ctl reconfigure命令写入的地方，存放应用数据和配置文件
/etc/gitlab omnibus 版本 gitlab的配置文件，平常只需要修改这个配置文件就够了
/var/log/gitlab 包含 omnibus 版本 gitlab 所有组件的日志数据

/var/opt/gitlab/nginx/conf 找到nginx配置，然后找到源码位置，修改了对 //opt/gitlab/embedded/service/gitlab-rails/app/views/events/event 去掉分享事件 Twitter widgets.js 的引用，直接删除第九行后面的代码
