mac上配置svn服务器的步骤

mac上面初始已经安装了svn服务端

1.建立目录

cd ~/Documents
mkdir svn-repos

2.创建仓库

svnadmin create svn-repos/account

3.配置登录用户

此时的仓库匿名是可以访问的，通常会配置用户后，在使用

编辑密码配置文件 

vim ~/Documents/svn-repos/account/conf/svnserve.conf

以下内容取消注释

anon-access = read
auth-access = write
password-db = passwd
authz-db = authz
realm = local-svn-repo

编辑权限配置文件

vim ~/Documents/svn-repos/account/conf/authz

配置示例

新建组manage 和 组 user，新建用户favoorr test，对目录 / ，manage组可以读写

[groups]
# harry_and_sally = harry,sally
# harry_sally_and_joe = harry,sally,&joe
manage = favoorr
user = test


[/]
# harry = rw
# &joe = r
# * =
@manage = rw

[repository:/svnroot]
# @harry_and_sally = rw
# * = r
manage = rw

编辑设置密码文件

vim ~/Documents/svn-repos/account/conf/passwd

示例

[users]
# harry = harryssecret
# sally = sallyssecret
favoorr = 000000

4.后台启动svn服务

svnserve -d -r svn-repos --listen-host 0.0.0.0

5.命令行导入文件

下面命令会把当前目录下所有文件上传到svn 仓库中

提交到svn仓库
svn import -m "initial import" . svn://127.0.0.1/account/trunk --username favoorr --password 000000

由于我们的仓库在本机，也可以使用以下命令提交到本机
svn import -m "initial import" . file:////Users/User/Documents/svn-repos/account/trunk --username favoorr --password 000000

6.命令行查看文件列表

svn list svn://127.0.0.1/account/trunk

后台查看进程

ps -ef |grep svnserve

三种重启Finder的方法

问题描述

1.今天在使用Sublime text的时候,经常使用的 Reveal In Finder的功能不能使用

2.昨天在Finder中使用 Go2Shell 的时候，打开Iterm非常慢

然后看了监视器，Finder占用350M多内存，本来想直接重启机器，突然想了下
应该是Finder有问题了，需要重启，就Google了下，总共收集3种快速重启方法

三种方法

方法一

摁 option 之后，右键点击dock上的Finder图标，会在最下面出现 重新开启 的菜单

方法二

在命令行终端执行 killall Finder

方法三

option + command + esc ,调出小窗口，找到Finder，强制退出重启

Redis设计与实现读书笔记

发表于 2014-11-20   |   分类于 redis   |     |   阅读次数

《Redis设计与实现》——读书笔记终篇

截止前天，《Redis设计与实现》纸质书看完了第一遍，感觉本书
是一本好书，值得观看。通过本书了解了Redis的实现细节，然后最深的
感触是Redis的作者是一个数据结构和算法拿得出手的人，这点需要肯定，然后是本书
的作者也是一个思维细腻，描述一个东西的时候希望做到透彻、清晰的人。

个人感觉特别适合对Redis有一些简单的了解，然后需要使用Redis来完成项目开发的
人员，读了这本书之后，虽然没有设计太多的最佳实践，但是你会对Redis的特性有一个整体的了解，
知道通过Redis可以很方便的实现什么功能？实现某一个特性需要哪些配置？不同的配置会有什么结果？
这些问题在通读本书后都会有相应的了解，而且本书也很适合放在手边做设计取舍时候做参考，
至少我是这么干得

以下几点是个人感觉Redis里面比较值得去学习和掌握的

  1. 5中数据类型的操作命令，这个在本书作者提供的免费网站上可以很方便的查阅，包括各种命令的介绍、
    使用示例，以及版本差异都有清晰的介绍，连接地址 http://redisdoc.com
  2. Redis关于过期时间的处理命令，和不同的设置方式
  3. Redis主从的使用

目前有使用Redis的地方

  1. 做数据缓存服务器，Redis中已经实现的时间过期功能，可以让我们很方便实现这个功能
  2. 计数器，接口访问次数调用，如果直接在数据库里面做，数据库压力比较大，但是Redis就好多了
  3. 好友关系存储和根据好友关系做可能认识的人推荐
  4. list来实现队列
  5. k v映射存储
  6. k set ，存储和某个用户有某种关系的uid集合

使用Redis的方式

单机部署，做从备份，没有使用到其他的复杂功能

mac os x 设置 jenkins开机启动遇到的问题

设置命令

取消开机启动

sudo launchctl unload -w /Library/LaunchDaemons/org.jenkins-ci.plist

设置开机启动

sudo launchctl load -w /Library/LaunchDaemons/org.jenkins-ci.plist

我的问题描述

第一次安装jenkins之后，是可以开机启动的，但是由于开机占用400++M内存，就被禁掉了，
然后现在回头想恢复让其开机启动，死活不成功，折腾过程记录如下

查看 plist 文件找出脚本，手工启动

zsh /Library/Application\ Support/Jenkins/jenkins-runner.sh

会出现以下错误

十一月 19, 2014 4:13:10 下午 hudson.UDPBroadcastThread run
警告: UDP handling problem
java.net.SocketException: Can't assign requested address
    at java.net.PlainDatagramSocketImpl.join(Native Method)
    at java.net.AbstractPlainDatagramSocketImpl.join(AbstractPlainDatagramSocketImpl.java:178)
    at java.net.MulticastSocket.joinGroup(MulticastSocket.java:319)
    at hudson.UDPBroadcastThread.run(UDPBroadcastThread.java:76)

这个错误不需要理会

出现错误 

十一月 19, 2014 4:13:12 下午 javax.jmdns.impl.DNSIncoming readAnswer
警告: There was an OPT answer. Not currently handled. Option code: 65002 data: 6DE368DADD8543B8
十一月 19, 2014 4:13:12 下午 javax.jmdns.impl.DNSIncoming readAnswer

解决办法为，增加如下参数，升级1.634版，也没用这个问题

java -Dhudson.DNSMultiCast.disabled=true

/usr/bin/java -Dfile.encoding=UTF-8 -Dhudson.DNSMultiCast.disabled=true -jar /Applications/Jenkins/jenkins.war

恢复jenkins开机启动办法

1.删除文件

sudo rm /Library/LaunchDaemons/org.jenkins-ci.plist

2.重新安装pkg安装包，

mac版下载地址 http://jenkins-ci.org/content/thank-you-downloading-os-x-installer

通过以上步骤，我的问题解决，祝君好运

3.以上两步是一种解决方法，或者通过 brew install jenkins 重新安装jenkins

4.手工在mac上启动 jenkins

/usr/bin/java -Dfile.encoding=UTF-8 -Dhudson.DNSMultiCast.disabled=true -XX:PermSize=256m -XX:MaxPermSize=512m -Xms256m -Xmx512m -Djava.io.tmpdir=/Users/Shared/Jenkins/tmp -jar /Applications/Jenkins/jenkins.war --httpPort=8080

tail -f /var/log/jenkins/jenkins.log
