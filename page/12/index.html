第一次租住小区房的教训

发表于 2015-04-30   |   分类于 heart   |     |   阅读次数

讲讲第一次租住小区房的故事

坐标：深圳 某地铁站旁

开始

原来租住的单间，合同期14年4月到15年4月，结果14年10月，房东涨了120，跨入1500，房子贵得原因是有个大阳台，单间面积不足30，
再加上旁边工地太吵，实在是很吵，一直在施工，女朋友老妈要来深圳治病，房子就必须租2房了

经过

找了中介，在2个地铁站看了房子，第一个地铁站，看了2套房子，没有心仪的，然后到后面一个地铁站，继续找中介只看了一套房，
就租了，签了合同，交了押金和首月房租，以及中介费。

这个房子上一个租客比较复杂，不是其他的租客，是房东儿子，然后就得等他们搬走，速度也挺快，
但是卫生没搞，床垫也少了一个，问题就来了，这些在合同里面都没有写，还好房东还算人好，我说了之后，说卫生我找人搞，房东给100块
钱，床垫，房东买，我签收就好

可是哪，我同学帮忙搬家时，看到小区外面有滚动字幕，房租价格比我租的价格要少1000，也就是，我tm被坑大法了，整个人都不好了，房子优点客厅、2室都带窗口，光线很不错，有阳台，虽然2室60平也是不大，但是自己看中的点，满足了后，整体感觉都还好，一次花费了1w4++，
不过房子算是搞定了，只不过是不是被坑还不知道、

当时中介和房东的情形，没看出来是被坑，但是如果真高1000，感觉有些离谱了，中介和房东，完全在演戏，然后，我竟没看出来，难道平时看电视太少了吗

先写到这，后续结果确定再说吧，我还是不相信，自己被坑这么狠 …

把人想的太好了，平时买东西感觉别人花很多时间给自己讲解什么或展示，自己就感觉应该买下来或者付钱，这种心理以后不能有，他们以这个生存，达成这笔交易，这个是他们应该做的最基本的，没有什么其他的，自己有决定权，该不该要，不能被他们蒙蔽，要找不同的比较

教训

  • 做事情，不管多急，一定要比较，不要怕错过，错过了才是真的没缘分
  • 租房时，有个交房的过程，交房时，在给所有钱，签所有合同，别让自己被动，房子的卫生，很烦人的，比如厨房的卫生没给你搞
  • 租房时，里面的家具，要在租房时跟房东谈的时候定下来包含什么，不包含什么，免得到时候麻烦，比如少个床垫什么的
  • 看房时，找到自己最关心的，比如光线、阳台，然后房间小点别太小就没关系
  • 如果是压二付一，写押金条的时候，一个月是房租押金，一月是家具押金，要让房东分开写
  • 中介，是个很不同寻常的群体，跟他们打交道，多多小心

ubuntu-配置无密码登录

发表于 2015-04-20   |   分类于 ssh   |     |   阅读次数

ubuntu-配置无密码登录

给Ubuntu设置无密码登录，必须做的工作，有时会耗费时间，积累，下次节约时间

需求

主机免密码登录 elk1 和 elk2

第一步，主机添加host解析

192.168.137.251 elk1
192.168.137.252 elk2

第二步，elk1生成.ssh目录

ssh-keygen ,一路回车，执行后，生成本机的.ssh目录

第三步，打开elk1的ssh公钥保存文件，第一次配置需要打开，后续不需要

sudo vi /etc/ssh/sshd_config

AuthorizedKeysFile      %h/.ssh/authorized_keys 这一行，取消注释

第四步，elk1加入主机的公钥

cd ~/.ssh

vim authorized_keys 把主机的公钥粘贴进来，ok

第五步，重启 elk1 的 ssh 服务

/etc/init.d/ssh stop
/etc/init.d/ssh start

第六步，验证

主机输入

ssh elk1@elk1 成功无密码登录，ok

以上描述的是第一次添加，如果不是第一次添加，只需要第四步，就ok

zookeeper入门-安装和启动-关闭服务

发表于 2015-04-05   |   分类于 zookeeper   |     |   阅读次数

从 Paxos 到 ZooKeeper ，分步实一致性原理到实践 读书笔记

zk配置

tickTime=2000
dataDir=/tmp/zookeeper
clientPort=2181
initLimit=10
syncLimit=5
server.1=IP1:2888:3888
server.2=IP2:2888:3888
server.3=IP3:2888:3888

server.id 每一行，代表一台机器
需要在数据目录 dataDir 创建myid文件，里面写机器编号
集群中 zoo.cfg 应该使用 svn 或者 git 管理，确保每台机器上的文件都能够一致
id 的范围是 1 ~ 255

启动服务

使用自带的shell启动
sh zkServer.sh start

使用原始Java方式启动
java -cp zookeeper-3.4.6.jar:lib/slf4j-api-1.6.1.jar:lib/slf4j-log4j12-1.6.1.jar:lib/log4j-1.2.16.jar:conf org.apache.zookeeper.server.quorum.QuorumPeerMain conf/zoo.cfg

验证服务

telnet 127.0.0.1 2181

stat

停止服务

sh zkServer.sh stop

zk可执行脚本

zkCleanup.sh 清理 zookeeper 的历史数据，包括事务日志文件和快照数据文件
zkCli.sh ZooKeeper 的简易客户端
zkEnv.sh 设置 ZooKeeper 的环境变量
zkServer.sh ZooKeeper服务的启动、停止、重启脚本
