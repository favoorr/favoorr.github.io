《Redis设计与实现》- 数据结构与对象

发表于 2014-10-29   |   分类于 redis   |     |   阅读次数

《Redis设计与实现》——读书笔记一

这本书从1.0开始一直关注，然后平时redis doc也是看的书的作者维护的网站，
可以预定后，立即预定，并成为第一批到手的读者，可惜中间由于各种原因一直
没有时间仔细研读，最近终于有时间可以来细细品读本书。

在地铁上看了本书的第一二三章，了解到本书的目录安排为四部分，分别是

  • 数据结构与对象
  • 单机数据库实现
  • 多机数据库实现
  • 独立功能的实现

第二章 简单动态字符串

讲解了设计理念，通过通俗易懂的语言和实例，讲解了sds(simple dynamic string)与
c语言原声的字符串之间的联系和区别，讲解了sds的设计结构，以及结构的优点

其中保存字符串len，该理念可以用到后台设计中，比如保存用户充值记录时候，会保存
每一次操作的单个记录，也会按照每个用户的操作计算，每个用户产生一条记录，比如充值
总额，第一次充值时间，最后一次充值时间，最后一次充值金额等元素

或者比如设计到count计算的数据，不会到用到的时候才去计算，而是出现数据后，就回去计算，
用到的时候，直接读取值，但是对于修改频繁的操作，保存时候，需要慎重，例如淘宝上每个商品
的浏览量，数据修改过于频繁，如果保存数据库，数据库压力会太大，而这部分数据又
不要去过于精确和实时，需要在准确性、数据库压力上找到权衡点

第三章 链表

初步讲解了链表在Redis里面的实现方案，listNode结构和list结构，以及多态性，可以用于多种
数据结构，链表和链表节点的api，可以带领读者快速理解实现方案方案和优点

今天开始尝试编写自己的博客

—2015年01月09日添加

今天有个同学问我我的博客是怎么弄的，他在我的“教导”下今天注册了github账号，在学git，
然后问我他为什么没有博客，然后我想补充下我的博客由来

fork from https://github.com/89ao/89ao.github.io，真正苦逼的经历是这样子的

在互联网上闲逛的时候看到一篇推荐文章 http://github.thinkingbar.com/redis-use-situation/,进来后
被页面的简单吸引，那时候我已经在csdn开始自己写简单博客，知道用csdn写是什么感觉，不过也就写了
6，7篇

然后问题来了，我想自己也有一个这样的博客该多好啊，然后我当时和我同学一样子，不知道作者是怎么弄的，
然后当时都想联系作者问了，灵机一动，Google神器，关键字 “github 博客”，然后首页的文章就惊呆了，原来
这东西早就很流行了，只是自己不知道而已 ~囧，

然后了解了下，啃了下英文资料，github为开发者提供的平台，github pages，可以来编写wiki等，而且对markdown已经支持完美，这里感谢github

然后的事情就是我找一个自己喜欢的模板，前端设计，就算了，我不擅长

然后继续万能的Google，找了找，找了找，感觉这个算简洁，

就fork了，修改了下，

原版的颜色有些受不了，太难看，整体设计不行，修改颜色还是没问题的

—2015年01月09日 end
