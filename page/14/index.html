Git 修改远程仓库

之前方法太笨，最新方法如下，直接替换即可

git remote set-url origin git@172.16.10.9:ott/blog.git

Git的远程仓库可以关联多个，而且可能遇到地址修改的需求，
例如前几天遇到需要将项目从自己目录，转移到小组目录里面

git remote --help 之后，看到了个newurl oldurl 的命令,想想应该就是它了

于是执行

git remote set-url origin git@172.16.10.9:ott/blog.git git@172.16.10.9:youname/blog.git

另外一种笨方法，删除之后，和新地址关联

git remote remove origin

git remote add origin git@172.16.10.9:ott/blog.git

git push --set-upstream origin master

Git水很深，高级用法很多，一点一滴积累

Git和Github、Gitlab很好用，没用的公司，赶紧用起来

graphviz 一个可以用文字画图的工具，非常强大好用

打开很慢

下面进入正文，想使用plantUML，需要安装graphviz，来使用dot命令支持plantUML的所有特性，
详细见plantuml官网，这里记录mac ox x10.10 安装时候遇到的问题

简单看了下官网，使用brew是可以直接安装的，但是过程中有些小问题，libpng的链接地址改变了，导致下载报错

中间遇到错误，看了下libpng这个包路径已经改变，所以

brew edit libpng

将地址

https://downloads.sf.net/project/libpng/libpng16/older-releases/1.6.13/libpng-1.6.13.tar.xz

然后继续

git实战篇--将本地项目提交到github/gitlab中

git实战篇–将本地项目提交到github/gitlab中

准备工作:

在github/gitlab中新建项目zhunions

1.在本地目录zhunions目录下初始化本地仓库

git init

2.建立.gitignore README.md文件

3.本地提交

git add . 提交当前目录
git commit -m "first commit"

4.添加远程关联

git remote add origin git@172.16.10.9:namespace/zhunions.git

5.本地分支与远程分支关联，并推送到远程仓库

git push --set-upstream origin master

以上5步可以完成本地仓库和分支与远程仓库和分支的关联.

执行所有操作后，有如下结果：

a.产生本地分支master

b.与本地仓库关联的远程仓库origin

c.与本地分支关联的远程分支master

d.本地代码全部提交到了远程仓库，其他人可以共享了

git rebase命令学习

阮老师对git rebase讲了一个点，可以和merge类似的操作，
最基本的时候，可以达到merge相同效果

一个台湾同胞写的教程，非常详细

http://blog.yorkxin.org/posts/2011/07/29/git-rebase

这里也简单介绍到了

http://gitbook.liuhui998.com/4_3.html

写下自己的理解：

假设

三个分支 master dev favoorr

场景一

favoorr衍生于 dev ，现在想修改到master，

可以在favoorr分支执行 git rebase master ，切换参考基准

场景二

favoorr 分支,本来衍生自提交aaaaa，现在想修改，衍生自提交yyyyy

在favoorr分支上上执行 git rebase –onto yyyyy aaaaa

git rebase –onto

git rebase –continue 如果有冲突，处理完冲突后，继续

场景三

1.开发的时候，遇到很多提交，想合并连续两次的小提交

2.或者之前的提交想改变顺序，提交zzz，应该放到提交xxx的前面

3.或者之前的提交commit写的有错别字，那么可以用rebase修改

git rebase -i xxxxx,你准备从这里开始整理提交，在提交一遍

git rebase 都可以做到

svn 里面没有搞过这种修改
