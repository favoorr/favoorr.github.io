学习知识的体会心得

目前自己掌握了基础的学习新知识的能力，在不断的被吊打中磨练出来的

知识，要积累，要多模仿，换成自己的，自己要有自己的知识体系和归档，方便用时候查阅

能力，需要不断的锻炼，不断去读书，打好基础，计算机行业的话，操作系统和数据结构和算法，这个
一定要懂，不要求十分精通，但是一定要掌握，不然永远是飘着的，根儿没扎好

学新东西，看官方，英文文档是很重要的，那么自然的，阅读英文的能力对于程序员来说是必备的，还不行的
话，就从现在开始，自己做项目的空闲时间，找自己当前项目用到的技术的官方reference或者document，提
高不是一蹴而就

学英文，stackoverflow要上，Quora要上，上面找自己感兴趣的话题和标签，他们会给你推送很多东西，读
然后进步

全面看一个东西的API很有用，比如学习Jquery，它的官方API如果你看过，而且看完了，肯定是会有收获的，

看书学东西，是很推荐的，但是刚开始的时候总是那么难，当你拿到一个600页，700页的书的时候，总是那么害怕，
当打开书的时候，一看内容，有些会，但有些真的不会，适当跳过，敲源代码，慢慢在工作中领悟

一本书都是作者智慧的结晶，写过博客的人才能体会到吧，不管人家写的多烂，或者是讲解API，毕竟肯定多多少少
加入了自己的思考和总结，总能学到自己不会得

遇到问题的时候，不轻易放过，多想想为什么自己想错了，当年理解Jquery对象和javascript对象之后，豁然开朗，为什么
对javascript对象.val()不能获取到值，而且后来教给了好几个同事

后台，分布式，多线程，tcp/ip很重要啊，工作5-6年如果感觉自己到瓶颈，那就是自己的问题了 ~~

14年一边看，一边学了Linux应用，因为没做过运维或管理员，很顺利的忘记了那些 囧~，但是我现在回来重新用的时候，
可以通过记忆和笔记不用Google解决问题，然后再分享成博客归档，我想我说我掌握Linux使用比较合适吧，千万不能说
精通，然后再走向熟练，但是也比了解好太多，只能说掌握，嗯，掌握，很好地中性词

靠搜索引擎才会的东西，不算掌握，只能说了解，而知识也都是自己亲身要用了，才会真正去学习，会有很多雷达，关注
很多东西，但是 没太走心

使用IDE的时候，把所有的提示都关闭了吧，然后API放手边，随时看，随时写，多写几遍，那些方法名就记下来了，然后
你会发现，自己在不断提高

早点建立自己的博客吧，也可以是csdn或者cnblogs，任何博客平台都好，关键是你要写，

14年，我在搜索引擎上发现可以用github写博客，当时很兴奋，但是一看早在2012年左右，人们已经用的如火如荼，好吧，又是别人已经不准备用的东西，我才知道，我才进来 在囧一次

当时，只记得那个同学告诉我，重要是你要写，在哪写不重要，这句话，永远记得，

最难的也就是，重要是你要做

尽管有jekyll，有markdown，有ruby，在当时都是一窍不通，但是都走过来了，

重要的是要做，步子迈起来是多么重要啊

要学得还很多

早点停止打游戏吧，庆幸11年下半年已经结束

时间总是挤的，也总是不够用

，嗯，早点睡吧，不早了 ~~~

竟然写了这么多

LInux实战--Mac配置ssh免密码登录到虚拟主机

LInux实战–Mac配置ssh免密码登录到虚拟主机

环境依赖

mbp系统：Mac os x 10.10.1
Linux： centos 6.6（可以参照这里下载ios镜像）
虚拟机软件：virtualbox4.3.20，使用帮助

写在最前

2014年3月份新购了Mac Book Pro 15，然后喜欢在上面使用VirtualBox安装虚拟机鼓捣东西,
最开始是鼓捣hadoop集群，由于13年花时间仔细阅读了Linux私房菜基础版，所以被轻轻虐待，
到最后能轻轻松给集群删除节点，配置新节点，当然，因为没有太多运维经验，所以是手工rsync+vim
虽然古老且笨拙，但是孰能生巧，中间好几个月搞其他东西，而且那时候也没有开始写博客，所以
东西都压在印象笔记里面，现在有新想法，搞gitlab + Java + Python + jenkins + test，而且
想玩自动化的，所以重新搞虚拟机，而且配置要求比较高点的。

这次一步一步，把所做的，都记录下来，下面回到正文：

主机的配置

首先在Mac上使用virtualbox安装虚拟机，详细可以参照博客文章

网络配置为：

主机跟虚拟机之间建立独立的局域网，    
主机ip 192.168.56.1     
虚拟机ip192.168.56.110,     
虚拟机hostname=centos65，    
主机和虚拟机都添加host域名解析

sudo vim /etc/hosts

新增
192.168.56.110 centos65

主机和虚拟机执行

ping centos65 都没有错误

主机生成ssh公钥和私钥

cd ~ && ssh-keygen 
然后一路回车

虚拟机配置

首先在虚拟机上生成ssh公钥和私钥

cd /root && ssh-keygen

编辑虚拟机ssh配置，开启ssh，此步骤不必须

vim /etc/ssh/sshd_config
 47 RSAAuthentication yes
 48 PubkeyAuthentication yes
 49 AuthorizedKeysFile      .ssh/authorized_keys

这三行取消注释
编辑后，退出保存

重启ssh服务

service sshd restart

验证ssh服务

netstat -tlnp | grep ssh

新增ssh认证文件，保存其他主机的公钥

vim /root/.ssh/authorized_keys

这时候，需要把mac上面的

cat ~/.ssh/id_rsa.pub

内容全部copy进来，不能有空格，保存退出

验证配置正确性

主机输入

ssh root@centos65

可能遇到问题

1.主机上~/.ssh/目录权限要是700
2.主机上~/.ssh/id_rsa权限要是600，并且属于当前用户

例如我的主机favoorr-mac,当前用户名User,

执行 ls -ld ~/.ssh/ ; ll ~/.ssh/ ，结果如下

favoorr-mac:~ User$ ls -ld ~/.ssh/ ; ll ~/.ssh/
drwx------  5 User  staff  170  5 14 23:44 /Users/User/.ssh/
total 24
-rw-------  1 User  staff  1679  5 14 23:44 id_rsa
-rw-r--r--  1 User  staff   398  5 14 23:44 id_rsa.pub

3.虚拟机上新建目录~/.ssh权限为644，文件~/.ssh/authorized_keys权限为644
4.可以发送主机上的id_rsa.pub到虚拟机上，然后cat id_rsa.pub >> ~/.ssh/authorized_keys
5.如果手工复制 id_rsa.pub 里面的内容，注意选中之后不要修改，直接粘贴，修改就可能报错了，权限那些其实不容易出问题，默认即可，这里比较容易出错

ssh -v root@dev1  调试 ssh 信息，比如本地多个私钥，
ssh -o GSSAPIAuthentication=no xxx@dev1

6.几个月后，一次爬坑经历发现，/etc/ssh/sshd_config里面的内容，是可以不注释的

Centos 6.5 实战-修改主机名

配置hadoop集群时，修改主机名，便于标示和记忆，记录如下

需要root权限，修改主机名为master步骤如下

第一步
vim /etc/sysconfig/network  实际修改主机名

添加
HOSTNAME=master

第二步
vim /etc/hosts  本地解析

添加本机 ip 和主机名映射
192.168.56.101 master

第三步，防止 ip 配置错误
ping master

第四步
reboot 重启

重启完成后，查看修改结果
hostname 查看主机名

mac上使用虚拟机网络配置总结

2014年初，公司让探索hadoop相关技术，一直以来有种习惯，不管什么技术，
必须自己玩下，然后感觉心里才有底，玩hadoop，单机是没啥意思的，分布式
就要多台机器，然后看了mactalk后，上面有段有了mac，不管你什么LVS、hadoop
想怎么玩就怎么玩，年初钱攒够就买了，之后就是mac新手，开始被吊打之路，研究hadoop，
过程是苦逼的，但是都走过来了，这会需要用虚拟机，想起来当时自己的苦逼，写出来
，记录下来。

根据mactalk的提示，我选择的虚拟机软件是virtualbox,然后过程中遇到问题，应该就是网络吧，首先说我的需求，

本机可以连接虚拟机
虚拟机可以连接本机
虚拟机可以上网
虚拟机之间可以ssh无密码登录
本机可以跟ssh无密码登录

看了不少文章，虚拟机网络NAT、桥接和Host only区别区别，个人理解如下

NAT

如果你的虚拟机eth0配置没有修改，即自动获取ip，那么虚拟机安装成功启动后，会自动获取的ip，而且获取的ip和宿主机是一个网段的，可以直接ping的通，而且可以联网,virtual box网络配置里面可以看到，command + ,在virtualbox网络那栏里面，可以看到10.0.2.0/24,然后虚拟机启动后，会自动生成ip 10.0.2.15

Host-Only

是在宿主机上新增一个网卡，然后在宿主机和虚拟机之间设置独立的局域网，然后宿主机和虚拟机沟通无障碍，
这里我遇到的问题是当时在虚拟机的网络配置页面，激活网卡二，选择了Host-Only后，下面的界面名称一直没办法选择，现在明白了也就是virtualbox 给主机要新增一个网卡，但当时不懂，愣是找了好久好久，自己都忘记了多久,同样command+` ，调出virtualbox网络设置页面，然后再virtualbox的网络配置页面新增网卡，然后再虚拟机的网络配置页面才有网卡可以选择，宿主机默认ip是192.168.56.1，最小ip是192.168.56.101，最大是192.168.56.254，什么东西都是会了不难，难了不会，添加网卡二之后，我的宿主机新增了个ip192.168.56.1，然后5台虚拟机ip是 192.168.56.101 ~ 192.168.56.105 ，一般同时启动3台虚拟机

桥接

会获取跟宿主机一个段的ip地址，比如宿主机ip 192.168.1.101 ,虚拟机会获取192.168.1.103的ip，但是公司的ip应该都是严格管理的，所以这种方法不好维护，如果主机所在局域网中得其他机器不需要使用虚拟机上的功能，最好使用Host-Only建立独立局域网

我的方案是

网卡一 NAT 方式和宿主机共享网络，虚拟机可以联网，方便下载安装各种软件
网卡二 与主机建立独立局域网，和路由上其他的机器分离，(现在看这种方式太对了，我后来把mac带到公司用了，因为要演示hadoop集群计算，而公司的ip是需要向网管申请的)
每一台虚拟机的ip固定，因为hadoop集群要设置master，ip要固定下来

网卡一使用NAT共享主机网络配置如下

cat /etc/sysconfig/network-scripts/ifcfg-eth0

DEVICE=eth0
HWADDR=08:00:27:41:90:4D
TYPE=Ethernet
UUID=1c2f383c-8fdb-4aa0-9e5b-8fa2f60117f9
ONBOOT=yes
NM_CONTROLLED=yes
BOOTPROTO=dhcp

新增网卡二，使用Host-Only方式配置如下

cat /etc/sysconfig/network-scripts/ifcfg-eth1

DEVICE=eth1
HWADDR=08:00:27:93:D4:33
TYPE=Ethernet
UUID=8ac73766-a02b-4417-8f71-6762218b6bf8
ONBOOT=yes
NM_CONTROLLED=yes
BOOTPROTO=none
IPADDR=192.168.56.101
NETMASK=255.255.255.0

/etc/init.d/network restart  重启网络配置
ifup eth0 /ifdown eth0  启用/停用某个网卡

难点总结

理解virtualbox给本机添加网卡
ip获取手工指定，修改默认的自动获取
