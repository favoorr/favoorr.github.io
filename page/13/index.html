jenkins使用maven构建时遇到的repository的错误

部署jenkins后，构建maven项目，在执行 -X mvn clean installl 时，报错内容如下

Started by user anonymous
Building in workspace /Users/Shared/Jenkins/Home/jobs/account2/workspace
Updating svn://127.0.0.1/account/trunk2 at revision '2014-11-24T11:52:51.614 +0800'
At revision 26
no change for svn://127.0.0.1/account/trunk2 since the previous build
[workspace] $ /Users/User/soft/apache-maven-3.2.1/bin/mvn clean install
[ERROR] Could not create local repository at /var/empty/.m2/repository -> [Help 1]
[ERROR] 
[ERROR] To see the full stack trace of the errors, re-run Maven with the -e switch.
[ERROR] Re-run Maven using the -X switch to enable full debug logging.
[ERROR] 
[ERROR] For more information about the errors and possible solutions, please read the following articles:
[ERROR] [Help 1] http://cwiki.apache.org/confluence/display/MAVEN/LocalRepositoryNotAccessibleException
Build step 'Invoke top-level Maven targets' marked build as failure
Finished: FAILURE

原因

重点看这句

[ERROR] Could not create local repository at /var/empty/.m2/repository -> [Help 1]

配置maven时，使用了默认的配置，所以就是匿名用户去寻找，

经验

1.配置maven时，不能使用默认配置，指定 setting.xml 的位置

2.setting.xml 配置中，尽量不使用 类似 ${user.home}/.m2/repository 使用环境变量的路径，
如果当前用户改变，整个本地仓库路径就改变了，

例如使用 jenkins 用户，路径就变为 /Users/Shared/Jenkins/.m2/repository
尽量使用 /Users/User/.m2/repository 这种绝对路径

mac 配置svn服务器

发表于 2014-11-22   |   分类于 svn   |     |   阅读次数

mac上配置svn服务器的步骤

mac上面初始已经安装了svn服务端

1.建立目录

cd ~/Documents
mkdir svn-repos

2.创建仓库

svnadmin create svn-repos/account

3.配置登录用户

此时的仓库匿名是可以访问的，通常会配置用户后，在使用

编辑密码配置文件 

vim ~/Documents/svn-repos/account/conf/svnserve.conf

以下内容取消注释

anon-access = read
auth-access = write
password-db = passwd
authz-db = authz
realm = local-svn-repo

编辑权限配置文件

vim ~/Documents/svn-repos/account/conf/authz

配置示例

新建组manage 和 组 user，新建用户favoorr test，对目录 / ，manage组可以读写

[groups]
# harry_and_sally = harry,sally
# harry_sally_and_joe = harry,sally,&joe
manage = favoorr
user = test


[/]
# harry = rw
# &joe = r
# * =
@manage = rw

[repository:/svnroot]
# @harry_and_sally = rw
# * = r
manage = rw

编辑设置密码文件

vim ~/Documents/svn-repos/account/conf/passwd

示例

[users]
# harry = harryssecret
# sally = sallyssecret
favoorr = 000000

4.后台启动svn服务

svnserve -d -r svn-repos --listen-host 0.0.0.0

5.命令行导入文件

下面命令会把当前目录下所有文件上传到svn 仓库中

提交到svn仓库
svn import -m "initial import" . svn://127.0.0.1/account/trunk --username favoorr --password 000000

由于我们的仓库在本机，也可以使用以下命令提交到本机
svn import -m "initial import" . file:////Users/User/Documents/svn-repos/account/trunk --username favoorr --password 000000

6.命令行查看文件列表

svn list svn://127.0.0.1/account/trunk

后台查看进程

ps -ef |grep svnserve

三种重启Finder的方法

问题描述

1.今天在使用Sublime text的时候,经常使用的 Reveal In Finder的功能不能使用

2.昨天在Finder中使用 Go2Shell 的时候，打开Iterm非常慢

然后看了监视器，Finder占用350M多内存，本来想直接重启机器，突然想了下
应该是Finder有问题了，需要重启，就Google了下，总共收集3种快速重启方法

三种方法

方法一

摁 option 之后，右键点击dock上的Finder图标，会在最下面出现 重新开启 的菜单

方法二

在命令行终端执行 killall Finder

方法三

option + command + esc ,调出小窗口，找到Finder，强制退出重启
